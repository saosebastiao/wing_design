# Wing Design — Development Plan

## Goal

A free-rotating, unstayed solid carbon-fiber **wingsail** built as a **space-frame of unidirectional CFRP beams** (pultruded straight rods, RTM-formed curved beams) wrapped and bound by a **filament-wound CFRP skin** that simultaneously forms the airfoil surface. Internal beam layout is generated by **structural optimization** (stress-aligned curved members + joint relocation + discrete cross-section sizing) and constrained at every step to remain manufacturable by **highly automatable composite processes**: pultrusion, filament winding, RTM-for-beams, and (sparingly) compression-molded CFRP "forging" for couplings.

The repo is meant to evolve from a 5 m demo wingsail to a process applicable from ~1 m FPV drone wings up to >100 m wind turbine blades.

## Working specification (first design point)

| Parameter           | Value                                         |
| ------------------- | --------------------------------------------- |
| Span                | 5.0 m                                         |
| Root chord          | 1.0 m                                         |
| Tip chord           | 0.6 m                                         |
| Airfoil             | NACA 0018 (symmetric, t/c = 0.18)             |
| Spar / pivot base   | 0.75 m below root, free 360°+ rotation        |
| Pivot location      | ~25% chord (passive feathering, refine later) |
| Material (primary)  | UD carbon / epoxy (E1 ≈ 130 GPa, anisotropic) |
| Manufacturing menu  | Pultrusion · RTM-for-beams · Filament winding · CFRP forging (couplings only) |

## Phases

Each phase ends in a runnable example under `examples/` and a passing smoke test. We deliberately spike end-to-end at low fidelity before deepening any one phase.

### Phase 1 — Wing geometry (done as scaffold)

- `wing_design.geometry.airfoil` — NACA 4-digit symmetric coords.
- `wing_design.geometry.wing` — lofted build123d solid + spar stub.
- `examples/01_wing_solid.py` — exports STEP/STL.

**Deliverable: ✅ STEP file of the 5 m wingsail.**

### Phase 2 — Aerodynamic loads (lean on AeroSandbox)

AeroSandbox gives us almost everything for free:

- `aerosandbox.Wing` mirrors our build123d spec directly (`WingXSec` per station with chord/twist/airfoil — NACA0018 is built in).
- `AeroBuildup` (fastest, subsonic, fully differentiable) for envelope sweeps; `LiftingLine` for nonlinear viscous fidelity; `VortexLatticeMethod` for span-resolved panel forces.
- `Atmosphere` + `OperatingPoint` for the conditions (apparent wind speed/angle as α with V≈AWS at sea level).

The one gap is **panel → FEA-mesh projection**: VLM/LL store per-panel `forces_geometry`, but we have to map those forces onto our build123d surface mesh ourselves. That lives in:

- `wing_design.aero.model` — thin wrapper that builds the ASB `Wing` from a `WingSpec`.
- `wing_design.aero.cases` — load-case envelope (design AWA/AWS sweep, gust, fully feathered, knockdown).
- `wing_design.aero.loads` — extract per-panel + spanwise loads from an ASB result and project onto the OML mesh.

**Deliverable: per-load-case nodal traction field on the wing surface, sanity-checked against analytic lift for an AR ≈ 7 wing.**

### Phase 3 — Coupled aero-structural baseline in ASB Opti

ASB ships `TubeSparBendingStructure` (tapered Euler-Bernoulli cantilever, isotropic-equivalent) and the differentiable `Opti` stack. That is exactly enough to do an end-to-end **coupled** baseline: jointly size a tapered tube spar + skin under the worst-case ASB-computed loads, with mass as the objective and stress / tip-deflection / buckling as constraints. The composite shell can enter as an isotropic-equivalent E with a knock-down factor for first pass; anisotropic-correct sizing comes in Phase 8.

- `wing_design.materials.unidir` — UD ply database (E1/E2/G12/ν12/strengths) + Classical Laminate Theory.
- `wing_design.structural.beam` — wraps `TubeSparBendingStructure` (or a box-beam analog) into an `Opti` problem with our load cases.

**Deliverable: total mass / max stress / tip-deflection envelope for the 5 m wingsail. This is the number every later phase has to beat.**

### Phase 4 — Volumetric FEA → stress tensor field

- `wing_design.structural.mesh` — gmsh tet mesh of the wing OML with the spar attachment as a Dirichlet region.
- `wing_design.structural.fea` — linear elastic 3D solve under each load case, isotropic-equivalent properties to start (Arora's approach). Produces σ(x) per tet plus Loubignac-smoothed nodal stresses.

**Deliverable: σ(x) field exported per load case, ready for frame-field fitting.**

### Phase 5 — Stress-aligned frame field + parametrization (Arora et al.)

- `wing_design.truss.frame_field` — fit R(x) ∈ SO(3) per vertex via the Rayleigh-quotient-induced norm + Laplacian smoothness. Reference implementation in MATLAB at `rarora7777/VolumetricTruss`; we re-implement the core pieces in Python.
- `wing_design.truss.parametrization` — solve the Poisson-like system for φ(x) (∇φ aligned with R).
- `wing_design.truss.extract` — trace integer isocurves of φ → three families of curves = the curved internal beam centerlines.

**Deliverable: a wireframe of curved beams inside the wing, viewable in PyVista / ocp_vscode.**

### Phase 6 — Manufacturability-constrained sizing (Jiang et al.)

- `wing_design.truss.alp` — Alternating LP: Algorithm a (force-density LP for fixed joints) + Algorithm b (linearized joint relocation LP), multi-load-case formulation.
- `wing_design.truss.manufacturability`:
  - **Pultrusion filter**: classify nearly-straight members; group co-linear neighbors into single pultruded stock pieces; constrain to a discrete catalog of cross-sections.
  - **RTM filter**: enforce per-member maximum curvature (min radius) and "approximately planar" constraints for curved beams.
  - **Joint regularization**: cluster joints within a clustering tolerance; tag joints needing CFRP-forged couplings vs. pure winding-bound joints.
  - **Stability post-pass**: Kassimali external-stability count from Jiang §6.2; add bracing members where needed.

**Deliverable: a sized truss whose every member is tagged with a manufacturing process and a stock cross-section.**

### Phase 7 — Filament winding path planner

- `wing_design.manufacturing.winding` — given the skin OML + the joint list, generate continuous winding passes that (a) form the airfoil surface and (b) wrap the joints / known buckling-prone members (IsoTruss-style reinforcement at nodes).
- Output: G-code or robot path stream, plus per-pass fiber orientation map for CLT.

**Deliverable: animated winding sequence; CLT-homogenized skin stiffness map.**

### Phase 8 — Verification + fixed-point iteration

- Re-mesh the **as-designed** structure (skin shell elements with the homogenized anisotropic stiffness from the winding plan, beam elements for the internal frame).
- Re-solve FEA under every load case; check stress, deflection, buckling, fatigue (later).
- Feed the new σ(x) back into Phase 5; iterate to convergence.

**Deliverable: certified-on-paper wingsail with documented load envelope and safety margins.**

### Phase 9+ — Scaling and other use cases

- Parameterize span / chord / taper to swap between wingsail / FPV drone wing / turbine blade.
- Add fatigue, dynamic (flutter, vortex shedding), and impact load cases.
- Hardware bring-up: 1 m test wing, bench loading.

## Decisions log

| Decision                                  | Choice                                                                                   |
| ----------------------------------------- | ---------------------------------------------------------------------------------------- |
| Build-out strategy                        | **Hybrid**: Phases 1–3 to depth (trustworthy aero + coupled-beam mass baseline), then spike Phases 4–7 at low fidelity. |
| Phase-4 FEA backend                       | **Roll our own linear-tet solver** in numpy/scipy for the spike pass. Promote to sfepy or FEniCSx when anisotropic homogenization starts mattering. |
| Phase-5 stress-aligned truss              | **Stress-line tracing first**: per-tet principal direction → boundary-seeded streamline integration → sampled truss. Faithful Arora port queued as a follow-on. |
| Skin role (early phases)                  | **Fairing only**: internal frame carries all loads, skin is mass-only. Structural-shell upgrade deferred to Phase 8.                                       |
| Phase-3 sizing stack                      | AeroSandbox `LiftingLine` + `TubeSparBendingStructure` jointly inside one `Opti` problem, isotropic-equivalent CFRP with a knock-down factor.              |
